# Skill: Audit Google Ads cho Doscom

## Mục tiêu
- **Lợi nhuận ≥30%** (KHÔNG dùng ROAS)
- **Profit** = Doanh thu − VAT 10% − Giá vốn − Chi phí QC
- **Trần CPA Search** = 0.6 × Giá bán − Vốn − VAT
- **Trần CPA Display** = 0.5 × Giá bán − Vốn − VAT

## 8 nhóm chấm điểm

| Nhóm | Trọng số | Thang điểm |
|------|----------|------------|
| Tracking | 25% | 80 nếu có conv tracking đủ; 50 nếu thiếu match Pancake; 30 nếu thiếu hoàn toàn |
| Profit/SP | 22% | Margin >30% → 80+; 0-30% → 50; lỗ → 20-40 |
| Lãng phí | 13% | CTR <1% hoặc CPC quá cao → 30-50; bình thường → 60-70 |
| RSA quality | 12% | Theo ad count + headline diversity |
| Cấu trúc KW | 10% | Theo phân tier search terms; Tier 3 nhiều = lãng phí |
| Trang đích | 8% | 50 mặc định + cần GA/heatmap data |
| Budget | 5% | Cân bằng giữa nhóm SP → 70+; quá tập trung → 30 |
| Compliance | 5% | 70 mặc định nếu không có flag |

## Grade

- 85+ = A
- 70-84 = B
- 55-69 = C
- 40-54 = D
- <40 = F

## Quy tắc

- Mỗi note ≥ 8 từ, có số liệu cụ thể nếu có data.
- summary ≥ 2 con số cụ thể (đ, %, tên SP/campaign).
- top_findings: 3 finding khác nhau, có action + số tiền + tên kw/campaign.
- total_score = round(sum(score×weight)/100). KIỂM TRA lại.
- Thiếu data thật sự → score = 35 (KHÔNG 0) + note "Thiếu data X".
